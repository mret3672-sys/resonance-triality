## GSSM -- Guide, Support, Stabilize, Mark

The building loop. OCTO reads the field; GSSM acts on it.

| Phase | Function | Direction |
|-------|----------|-----------|
| **Guide** | Set trajectory | Travels left (navigational) |
| **Support** | Provide resources | Holds position |
| **Stabilize** | Lock gains | Prevents regression |
| **Mark** | Record/anchor | Travels right (depositional) |

### Inverse Reading (MSSG)

```
Mark --> Stabilize --> Support --> Guide
```

This is the diagnostic direction: where are we now? --> what's holding? --> what's supporting? --> where were we headed?

### GSSM and the 132/231 Mapping (STATUS: PROBABLE, Cf=0.75)

The original mapping placed GSSM on 132: Guide=Ideal, Support+Stabilize=Boundary, Mark=Core. But GSSM's origin in plumbing challenges this.

**Physical trade reality:** You don't start at the ideal design and navigate inward -- you start at the pipe in your hands (Core/2), work through resistance (Boundary/3), and build toward the designed system (Ideal/1). That sequence is **231**, not 132.

**Emerging reframe:**

| Mapping | Direction | Sequence |
|---------|-----------|----------|
| OCTO + 132 | Diagnostic (read inward) | Ideal --> Boundary --> Core |
| GSSM + 231 | Generative (build outward) | Core --> Boundary --> Ideal |

**GSSM remapped on 231:**
- **Guide** = start at Core (2) -- you guide FROM where you actually are
- **Support + Stabilize** = push through Boundary (3) -- resources against resistance, locking gains
- **Mark** = arrive at Ideal (1) -- stamp the completed structure

Position 3 (Boundary) remains the middle passage in both directions. This is why 213 was correctly demoted during the epistemic audit -- it skips Boundary entirely.

### Reception Protocol

```
Believe --> Antenna --> Received
```

GSSM builds outward (Guide-->Mark). This is its complement -- the inward reception sequence:
- **Believe** sets the tuning (what you're open to receiving)
- **Antenna** is the active state of receptivity (positioned and oriented)
- **Received** is the moment signal lands

GSSM is construction (exhale). Reception is capture (inhale). Together they form the full breathing cycle.

## The 132/231 Counter-Rotation (STATUS: PROBABLE, Cf=0.75)

132 and 231 aren't sequential modes you switch between. They're simultaneous -- two perspectives traversing the same Boundary from opposite directions.

### The Geometry

| | 132 (Diagnostic / Guide) | 231 (Generative / Navigator) |
|------|--------------------------|------------------------------|
| **Start** | Ideal (1) -- already held, already in mind | Core (2) -- already moving, existing momentum |
| **Middle** | Boundary (3) -- the shared pressure point | Boundary (3) -- the shared pressure point |
| **End** | Core (2) -- what's actually true | Ideal (1) -- aspirational, what they're moving toward |

Position 3 is the convergence point. Both traversals pass through Boundary -- one traveling inward (diagnosis), one traveling outward (construction).

### Phase Lock Mechanism

When the Ideal (1) is uttered, it plants itself in the other participant's awareness. Pressing hard on Boundary (3) doesn't just create friction -- it activates imagination. Now there's a gap between what was planted (1) and what resists (3). That gap generates a desire for resolution that belongs to the participant, not the guide. Core (2) is where resolution locks -- the standing wave stabilizes.

**Critical property:** Phase lock doesn't require completion. Even without a transaction or agreement, the emotional association between the uttered Ideal and the felt Boundary pressure persists. That's Mark without close -- the GSSM cycle deposits a structural artifact in the participant's field simply by engaging the 132 architecture.

**Domain independence:** Any time 132 runs against a conscious participant -- teaching, preaching, mentoring, even conversation -- the same phase lock mechanism applies. The specific domain changes. The geometry doesn't.
